# 仓库规则入口

本目录存放当前仓库的开发规则。修改 C/C++ 代码前必须先读本文件。

## 必读规则

- [C++基础编码规范](./cpp-style-guide.md)

## 使用方式

1. 先观察待修改文件附近的现有风格。
2. 再阅读 `cpp-style-guide.md` 中与本次修改相关的章节。
3. 新增或重写代码时按规范执行；小范围修改时同时保持局部代码一致性。
4. 规范与用户当次明确指令冲突时，以用户指令为准，并继续遵守未冲突部分。

## C/C++ 修改前检查

- 命名：文件、类、函数、变量、成员变量、宏是否符合规范和局部惯例。
- 头文件：包含保护、前置声明、包含顺序是否合理。
- 类设计：构造、拷贝、继承、访问权限和友元是否清晰。
- 实现：函数长度、参数、异常、宏、成员访问是否符合规范。
- 注释：只在必要处解释意图，复杂预处理分支需要结尾注释。
- 格式：缩进、空格、括号、对齐方式跟随规范和邻近代码。
- 内存：优先使用静态或栈对象，堆对象交给智能指针、Qt 父子对象或容器管理。
